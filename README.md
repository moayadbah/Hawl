# حَوْل — Hawl

منصة تربط حساباتك البنكية المتعددة عبر المصرفية المفتوحة السعودية، تتتبّع بلوغ
النصاب واكتمال الحول الهجري عبر **آلة حالات (Finite State Machine)**، تحسب الزكاة
بدقة، وتتيح دفعاً محاكى جاهزاً للتكامل مع PIS.

**🔗 التطبيق المباشر:** https://hawl-team.vercel.app

**📊 العرض التقديمي:** https://docs.google.com/presentation/d/1G9sJNrV1KNP_MEVMOn3bBqHopcVCqc12eCoTQnlxWY4/edit?usp=sharing

## التشغيل محلياً

```bash
npm install
npm run dev
# افتح http://localhost:3000
```

## متغيرات البيئة (اختيارية)

المشروع يعمل بالكامل بدون أي إعداد. لتفعيل حفظ الجلسة/تسجيل الدخول وربط البنوك
وسعر الفضة اليومي المباشر بشكل دائم (بدل الاعتماد على النسخة المحلية المخزّنة)،
انسخ `.env.local.example` إلى `.env.local` وعبّئه بمفاتيح مشروع Supabase الخاص بك:

```bash
cp .env.local.example .env.local
```

## أوامر مفيدة

```bash
npm run dev          # خادم التطوير
npm run build        # بناء للإنتاج (جاهز لـ Vercel)
npm run gen:data      # إعادة توليد بيانات البنوك التجريبية (AIS-format)
npm run engine:check  # التحقق من آلة الحالات (يجب أن تعطي الزكاة 1,208.40)
npm run fetch:silver  # جلب تاريخ سعر الفضة الحقيقي وتحديث data/silverPriceHistory.json
npm run seed:silver   # تخزين سعر الفضة الحالي في Supabase (يحتاج .env.local)
```

## آلية حساب الزكاة

### النصاب: متغيّر يومياً وليس رقماً ثابتاً

النصاب المعتمد هو **نصاب الفضة (595 غرام)**، وهو الأحوط والأكثر شيوعاً في الفتاوى
المعاصرة على الأموال النقدية. بدل تثبيت قيمته بريال ثابت، يُعاد حسابه **كل يوم** من
سعر الفضة الحقيقي في السوق العالمية (595 × سعر الجرام بالريال في ذلك اليوم بالذات)،
عبر بيانات تاريخية حقيقية مجلوبة من السوق (`data/silverPriceHistory.json`، ويُحدَّث
بالأمر `npm run fetch:silver`)، مع تخزين مؤقت اختياري لسعر اليوم الحالي عبر Supabase.

### تتبّع الحول: آلة حالات (وليس خط زمني بسيط)

يتتبع التطبيق **رصيداً نقدياً يومياً** (تجميع أرصدة كل الحسابات المربوطة) عبر آلة
حالات منتهية بخمس حالات، والانتقال بينها محكوم بقاعدة واضحة لكل خطوة:

```
دون النصاب ─[بلغ النصاب]→ بدأ الحول ─[مرّ يوم فوق النصاب]→ الحول جارٍ
                                                                │
                    ┌───────────────────────────────────────────┤
          [نزل دون النصاب]                          [تمّت سنة هجرية كاملة]
                    ▼                                           ▼
               انكسر الحول                                 الزكاة واجبة
```

أي نزول للرصيد دون النصاب في أي يوم أثناء الحول **يُلغي الحول الجاري** ويبدأ العدّ من
جديد عند بلوغ النصاب مرة أخرى — لا "تراجع" في الحالة، بل انتقال جديد إلى الأمام، وهذا
هو سبب اختيار آلة الحالات كأساس للمحرك.

### بعد وجوب الزكاة: طريقتا حساب للمال المُضاف لاحقاً

الأموال المُضافة إلى الحساب بعد اكتمال الحول الأول تحتاج طريقة واضحة للتعامل معها.
يطرح التطبيق طريقتين:

- **السنوية المبسّطة (الافتراضية):** حول واحد موحّد لكل الأموال — تُخرَج زكاة %2.5 من
  إجمالي الرصيد كاملاً، مرة واحدة في السنة، بدءاً من تاريخ اكتمال الحول الأول ثم كل سنة
  هجرية بعدها. المبلغ الواجب يتغيّر كل سنة تبعاً لنمو أو نقصان الرصيد.
- **الدقيقة — لكل مبلغ حوله الخاص:** الرصيد الأساسي يوم اكتمال الحول الأول يُزكّى كل
  سنة هجرية من ذلك التاريخ، وكل إضافة مالية لاحقة تُسجَّل كدفعة مستقلة لها حولها الخاص
  من تاريخ إضافتها بالضبط، ولا تجب زكاتها إلا بعد مرور سنة هجرية كاملة عليها بشرط بقاء
  الرصيد الكلي فوق النصاب. السحوبات تُستهلك من أحدث الدفعات أولاً (LIFO)، وأي مبلغ يُسحب
  قبل تمام حوله لا زكاة فيه. مناسبة أكثر لمن يريد دقة محاسبية تفصيلية.

## البنية

- `app/` — الواجهة (App Router)، بما فيها `app/api/*` (نقاط API: ربط البنوك، التحليل،
  الدفع، تسجيل الدخول/الضيف، وحفظ الإعدادات)، و`app/sources/` (دليل الطريقة بلغة مبسّطة).
- `lib/` — المحرّك: `stateMachine.ts` (آلة الحالات)، `fiqhRules.ts` (جدول الانتقالات
  ومصادرها)، `simEngine.ts` (طريقتا حساب الزكاة)، `hijri.ts`، `zakat.ts`، `analysis.ts`،
  و`server/silverPrice.ts` (سعر الفضة اليومي الحقيقي) و`server/supabase.ts` (تخزين
  اختياري لجلسات المستخدمين وربط البنوك والإعدادات).
- `data/` — 3 ملفات بصيغة AIS (الإنماء، الراجحي، الأهلي) تروي قصة «أحمد»، وتاريخ سعر
  الفضة الحقيقي.
- `components/` — الشاشات (تسجيل الدخول/الضيف، ربط البنوك، الأصول، سجل الحول، نتيجة
  الزكاة، الدفع، محاكاة الأعوام القادمة، الإعدادات) وواجهة المستخدم بهوية الإنماء البصرية.
- `design.md` — لغة التصميم المعتمدة (ألوان، طباعة، ظلال) — المرجع لقيم `tailwind.config.ts`.

## الهوية البصرية

يستخدم النموذج خط مصرف الإنماء الرسمي (`AlinmaTextTT`) وألوان هويته الجديدة،
بينما يحتفظ **حَوْل بشعاره الخاص**. شعار الإنماء يظهر فقط بجانب العناصر المتعلقة
بالإنماء (زر الربط، صف الحساب، زر الدفع).
